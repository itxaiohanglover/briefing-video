## 关键设计决策

### 1. 音频驱动时长

**问题**：固定时长配置导致音画不同步
**解决**：
- 先用 TTS 生成音频
- 用 ffprobe 检测实际时长
- 视频时长 = 音频时长总和

**优点**：
- 音画完全同步
- 支持任意语速调整
- 无需手动计算帧数

### 2. 5 场景模板

**问题**：完全自由的场景导致用户决策疲劳
**解决**：标准化 5 场景

| 场景 | 用途 | 时长 |
|------|------|------|
| Intro | 开场钩子 | ~6s |
| Slideshow | 图片展示 | ~12s |
| Subtitle | 文字叙述 | ~8s |
| Dashboard | 数据展示 | ~32s |
| Outro | 结尾 | ~8s |

**优点**：
- 降低用户认知负担
- 保持品牌一致性
- 易于优化每个场景

### 3. 图片 opacity 策略

**问题**：图片被过度遮罩导致消失
**解决**（参考玄清）：

| 场景 | opacity | 用途 |
|------|---------|------|
| Intro 分屏 | 0.9 | 视觉冲击 |
| 背景 | 0.55 | 氛围营造 |
| Outro | 0.4 | 弱化背景 |

**禁止**：
- opacity < 0.25
- 深色遮罩 + 模糊叠加

### 4. 按句字幕

**问题**：打字机效果阅读体验差
**解决**：按标点切分，均匀分配

```
原文：2024年数据显示：锂电产业产值达1050亿元，同比增长35%。

切分：
- "2024年数据显示："
- "锂电产业产值达1050亿元，"
- "同比增长35%。"

每句显示时长 = 场景总帧数 / 句子数
```

## 技术选型

| 组件 | 技术 | 理由 |
|------|------|------|
| 视频框架 | Remotion | React + TypeScript，代码即视频 |
| 动画 | Remotion spring/interpolate | 函数式，精确控制 |
| TTS | Edge TTS | 中文效果好，支持克隆 |
| 时长检测 | ffprobe | 行业标准，精确到毫秒 |
| 构建脚本 | Python | 简洁，适合 CLI 工具 |

## 扩展性

### 添加新场景类型

1. 创建 `templates/src/scenes/Scene06.tsx`
2. 在 `Root.tsx` 中添加 Sequence
3. 更新 `markdown-parser.ts` 支持新类型

### 添加新 TTS Provider

1. 实现 `TTSProvider` 接口
2. 在 `generate_audio.py` 中添加新 provider

### 自定义样式

修改 `config.json`：
```json
{
  "scenes": {
    "intro": {
      "imageOpacity": 0.95,
      "kenBurns": false
    }
  }
}
```
